# fn-4-4o2.6 Replace hardcoded values with CSS variables

## Description
Replace remaining hardcoded color values with CSS variables from the design system for consistency and maintainability.

### Problem

Despite having a good CSS variable system in `variables.css`, there are ~80+ hardcoded color values scattered across stylesheets:
- Button hover/active states: #dc2626, #b91c1c, #059669, #047857
- Wheel segment colors: #e74c3c, #3498db, etc.
- Onboarding step colors

This makes global color changes difficult and increases maintenance burden.

### Solution

1. Add missing color variables to `variables.css`:
   - `--danger-hover`, `--danger-active`
   - `--success-hover`, `--success-active`
   - Wheel segment colors as indexed variables

2. Replace hardcoded values with variable references

3. Exclude intentionally hardcoded values (e.g., third-party brand colors)

### Files to Modify

- `src/popup/styles/variables.css` - Add missing color tokens
- `src/popup/styles/components/buttons.css` - Use variables for hover/active
- `src/popup/styles/features/giveaway.css` - Wheel segment colors (optional - may keep for variety)
- `src/popup/styles/features/onboarding.css` - Step indicator colors

### Key Code Locations

- `src/popup/styles/variables.css:5-20` - Color definitions
- `src/popup/styles/components/buttons.css:59,63,74,78` - Hardcoded hover states
- `src/popup/styles/features/giveaway.css:340-347` - Wheel colors
